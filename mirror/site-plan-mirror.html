Content

Home page

My Mirror reflects on professional technical skills that I have and what I've been able to achieve with them. Future project partners and employers will need to know where I fit in their organisation and what I can do for them. Here is a portfolio that helps them figure that out very quickly and maybe even decide if I'm a good hire. There are two sections, one focused on my relevant technical skills in Software Engineering and the other is about what I've done in that field.

1. Technical Skills:

  • What the skill is.
  • How I developed the skill (Self-taught, Bootcamp or University?)
  • Proficiency level: For how long have I practised it and how comfortable do I feel with conducting projects using that skill.

2. Achievements:
  • Explanation of the project: Purpose and impact of the project.
  • How long it took to build it and the challenges invloved.
  • What I learnt and gained during that time: networks, more skills.

Technical Skills


  • A detailed description of programming languages that I am proficient in. It includes two major divisions:
    Front-end languages and Back-end languages.
  • Complementary skills in Cloud DevOps: This includes being a certified practitiober of platforms like Amazon Web Services and Microsoft Azure.
  • How did I build each skill? Self Taught (length of time), Bootcamp or University?

Images for the Technical Skills page



Achievements

A list of projects that I've undertaken, individually or with a team.
For each achievement section, the following questions will be answered:

  • Project description: Problem it is solving and how it does that.

  • Challenges that I faced and specific things that I did to overcome them.

  • Project Timeline: How long it took from inception to launching.

  • Results of the project: impact in society and how it impacted me as well. What did I learn and gain from the experience?

  • Status of the project: Is it dead or alive and why?
